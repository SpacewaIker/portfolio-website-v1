Mes projets

Une des meilleures façons d'apprendre la programmation est en pratiquant et en travaillant sur des projets. Que ce soit seul ou en équipe, pour un cours ou un projet personnel, essayer de créer quelque chose est crucial pour concrétizer les connaissances théoriques.

Mes plus gros projets sont présentés ci-dessous, et la majorité sont aussi présentés sur GitHub. Faites défiler pour les découvrir!

The Amulet of Sirref

Mai 2023 - Juil 2023
  • Rust
  • Bracket-lib
  • WebAssembly
  • Développement de Jeux Vidéos
  • A développé un jeu roguelike en Rust avec la librairie Bracket-lib et le ECS Legion.
  • A implémenté un système de génération de cartes procédural, un système de combat au tour-par-tour, et une IA d'ennemis simple.
  • A publié le jeu publiquement en application web assembly, jouable depuis virtuellement tous les appareils avec un clavier.

Pour ce deuxième projet en Rust, je voulais me lancer plus sérieusement dans le développement de jeux vidéos. J'ai donc décidé de suivre le livre Hands-on Rust de Herbert Wolverson, un tutoriel d'introduction sur les jeux roguelike en Rust. J'ai par contre poussé mon jeu beuacoup plus loin en ajoutant beaucoup de fonctionnalités telles différents modes de jeu, plus d'items, plus d'ennemis, etc.

Le jeu est un roguelike au tour-par-tour, où le joueur doit trouver l'Amulette de Sirref, un artefact puissant qui lui permettra de vaincre les monstres maléfiques. Le joueur doit se battre contre des monstres à travers le donjon, collectant des items pour l'aider sur sa quête, tels des armes, de l'armure, des potions et des items spéciaux.

Globalement, je suis très fier de ce que j'ai accompli pour mon premier projet sérieux en développement de jeux vidéos. J'ai aussi appris énormément à propos du développement de jeux vidéos, notemment comment fonctionne l'architecture entity-component-system, tout en améliorant mes capacités en Rust.

Le jeu est disponible en ligne, directement dans votre navigateur web: essayez-le! (clavier nécessaire).

Démo d'application web Rust

Jan 2023 - Avr 2023
  • Rust
  • Rocket
  • SeaORM
  • Yew
  • PostgreSQL
  • A développé une application web en utilisant le langage de programmation Rust pour le stack complet.
  • A utilisé le framework Rocket avec SeaORM pour le backend et Yew pour le frontend.
  • A créé un frontend d'application web vif, s'exécutant dans le navigateur grâce a WebAssembly.

Le but principal de ce projet était de jeter un coup d'oeil aux frameworks web en Rust et de voir comment serait l'expérience de créer une application web complètement en Rust. Le site web est une application de gestion de discographie, où l'utilisateur peut créer et lier des artistes, des albums, et des chansons.

Globalement, j'ai été impressioné par les frameworks, étant donné que Rust est un langage encore relativement jeune. Les frameworks étaient similaires à d'autres frameworks avec lesquels j'avais précédemment travaillé, tels Spring et React, tout en gardant les aspects qui rendent Rust génial. Je me suis beaucoup amusé à travailler sur ce projet, à travers lequel j'ai aussi appris beaucoup à propos de Rust et de WebAssembly.

Purposeful

Jan 2023 - Mai 2023
  • Java
  • JavaScript
  • Spring
  • Next.js
  • PostgreSQL
  • A développé une application web permettant aux utilisateurs de partager des idées de projets et de trouver des collaborateurs.
  • A travaillé dans une équipe de 10 développeurs pendant deux sprints pour développer une application complètement fonctionnelle.
  • A géré le développement du projet en utilisant le principe scrum avec un product owner et un scrum master.
  • A implémenté des fonctionnalités avancées telles l'autentification OAuth2 avec Spring Security et des tokens JWT.

Pour ce projet, nous nous sommes attaqués à un problème que nous voyons souvent en tant qu'étudiants en génie logiciel: tout le monde veut travailler sur des gros projets, mais tout le monde travaille seul, ce qui limite le potentiel de ce qui est créé. Le but était donc de créer une plateform où les utilisateurs peuvent facilement collaborer pour créer des projets ayant une utilité et un but concrets.

Une grosse partie du projet était aussi d'avoir l'expérience de travailler dans une grande équipe, avec 10 développeurs, et de voir comment gérer le projet avec la méthodologie scrum. Nous avons donc travaillé sur le projet pendant deux sprints de trois semaines, où nous gérions le backlog, précisions les tâches au début des sprints, se recontraient pour planifier les prints, etc. Nous avions aussi un product owner pour concentrer nos efforts dans le but d'avoir un projet fonctionnel et un scrum master qui aidait les autres développeurs s'ils avaient des problèmes.

Le résultat est probablement le projet le plus poli sur lequel j'ai travaillé avec beaucoup de fonctionnalités et une très bonne expérience utilisateur. Je suis très fier de ce que nous avons accompli, étant donné le laps de temps et nos horaires très occupés. Nous avons aussi pu travailler sur des fonctionnalités plus avancées telles l'autentification, ce qui est quelque chose que nous avions souvant négligé précédemment à cause de contraintes de temps.

Gestionnaire de Musée

Oct 2022 - Nov 2022
  • Java
  • TypeScript
  • Spring
  • Next.js
  • PostgreSQL
  • A créé une application web pour gérer les items, employés et clients d'un musée.
  • A développé une application web moderne du design du modèle de domaine et le backend Java avec Spring et une base de données PostgreSQL au frontend avec TypeScript et Next.js.
  • A collaboré au sein d'une équipe de six développeurs en utilisant des GitHub Issues et un GitHub Project pour gérer le développement.

Ce projet est une application web de gestion pour un musée fictif qui permet la gestion des items, employés et clients du musée. Les items peuvent être déplacés dans des pièces spécifiques ou en stockage et peuvent être loués par les clients. Les clients peuvent aussi donner des items au musée. Les employés peuvent approuver les locations et les donations ainsi que de mettre des items en stockage ou en salle de montre. Finalement, le propriétaire du musée peut performer les mêmes actions qu'un employé, mais peut aussi engager et renvoyer des employés.

Puisque ce projet est plus ou moins un successeur au projet DiveSafe, les buts sont similaires : d'apprendre des techniques de gestion de projet et de se familiariser avec les outils utilisés en industrie. Comme tel, nous avons utilisé un tableau Kanban avec un GitHub Project pour gérer les tâches. Nous avons aussi utilisé GitHub et des GitHub Issues pour collaborer sur le projet et pour gérer notre code. Le backend a été écrit en Java avec le framework Spring connecté à une base de données PostgreSQL. Le frontend a été écrit en TypeScript avec Next.js.

DiveSafe

Jan 2022 - Avr 2022
  • Java
  • MVC
  • UML
  • TDD
  • BDD
  • Gherkin
  • JavaFX
  • A développé une application avec interface graphique permettant de gérer les membres et les excursions de plongée, ainsi que la location d'équipement.
  • A collaboré avec cinq autres développeurs du design de l'architecture au développement et à la phase de tests.
  • A utilisé le motif d'architecture modèle-vue-contrôleur et a développé l'application en « test-driven-development » ainsi qu'en « behavior-driven-development ».
  • A designé et créé un logo pour l'application.

L'application DiveSafe est une application de gestion pour une companie de plongée fictive. Elle permet de gérer les membres du club, les guides de plongée et leur horaire, les excursions des membres, et la location d'équipement.

Le but principal de ce projet était d'expérimenter avec des motifs de design et des styles de développement populaires. Le motif d'architecture choisi était donc le modèle-vue-contrôleur. Le modèle a été construit à partir d'un diagramme UML, et les classes Java ont été générées automatiquement à partir d'une description du modèle grâce à l'outil Umple. Certaines parties du contrôleur ont été développées en « test-driven-development », et d'autres en « behavior-driven-development ». Pour les tests unitaires, nous avons utilisé JUnit, alors que pour les tests d'intégration, nous avons utilisé Cucumber et Gherkin. Finalement, nous avons utilisé JavaFX pour créer l'interface graphique.

J'ai aussi designé et créé le logo de l'application visible ci-dessus!

AirDraw

Oct 2021 - Nov 2021
  • Python
  • SciKit Learn
  • MediaPipe
  • IA/ML
  • Vision par ordinateur
  • StreamLit
  • A développé une application web basée sur l'intelligence artificielle en Python.
  • A créé, entraîné, et sauvegardé le modèle de machine learning utilisé pour la reconaissance des signes de mains.
  • A amélioré la performance du modèle en utilisant des techniques d'augmentation de données.

Ce projet était le dernier du cours d'IA « MAIS 202 » de la société d'intelligence artificielle de McGill. Le but du projet était d'utiliser tout ce que nous avions appris dans le cours pour créer une application basée sur l'intelligence artificielle. Mon coéquipier et moi étions intéressés par la vision par ordinateur et avons donc décidé de créer une application où l'utilisateur dessinerait des formes en bougeant sa main et en faisant des signes devant une caméra.

Nous avons géré le développement complet de l'application, de la collection des données d'entraînement au choix d'algorithme et la création et l'entraînement du modèle. Nous avons décidé d'opter pour un algorithme d'arbre de décision pour la classificiation des signes de main. Nous avons utilisé la librairie Hands de MediaPipe pour extraire la position des mains du flux vidéo. Pour l'application web nous avons utilisé StreamLit et PyVirtualCam puisque ces librairies étaient simples et correspondaient parfaitement à nos besoins.

Puisqu'il s'agissait d'un projet d'équipe, j'ai pu développer mes compétences de travail d'équipe. Notament, j'ai du démontrer de bonnes compétences de communication et de gestion du temps à cause de problèmes que j'ai rencontrés avec mon coéquipier. Le code pour l'entraînement du modèle ainsi qu'une preuve de concept sont disponibles sur le dépot GitHub. Une page DevPost a aussi été créée pour le projet.

Pong

Oct 2021 - Dec 2021
  • Java
  • Développement de jeux video
  • Simulation de physique
  • Programmation oritentée objet
  • A recréé le jeu de Pong en Java avec une simulation de physique pour le mouvement de la balle.
  • Le jeu inclut des options telles la difficulté de l'adversaire, la vitesse de la simulation, et les couleurs de la balle et des raquettes.

Pour ce projet, j'ai recréé le jeu classique de Pong en Java. Ceci était mon premier peojet en Java et une introduction plus formelle à la programmation orientée objet. Le jeu a donc été développé en premier lieu avec un style fonctionnel/impératif et a été plus tard converti en un style orienté objet. L'orienté objet permettait un maintien du code plus facile, ainsi que des fonctionnalités additionelles telles plusieurs simulations de balles avec du multithreading.

Le jeu inclut de nombreuses fonctionnalités : un systeme de pointage avec des joueurs que l'on peut renommer, des options de couleurs pour la balle et les raquettes, des options de jeux et des raccourcis clavier pour des actions comme récommencer, pauser ou quitter la partie.
