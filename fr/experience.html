Pour mon deuxième stage, j'ai eu la chance de travailer en tant qu'assistant de recherche au Reliable Silicon Systems Lab à l'université McGill. Ma tache était de configurer une carte de développement pour le machine learning et de trouver comment exécuter un modèle de machine learning dessus. La carte de développement était un Raspberry Pi avec un module de caméra doté d'une puce d'accélération matérielle pour les tâches de machine learning.

Pendant le stage, j'ai implémenté plusieurs modèles de computer vision provenant d'articles publiés, tels que LeNet et PortraitNet, en utulisant Tensorflow. Chaque modèle devait être quantifié en nombres entiers de 8 bits en utilisant Tensorflow Lite, avant d'être traité avec la chaine d'outils de la carte. Ensuite, une application C++ utilisait le modèle final pour être executé directement sur l'appareil périphérique.

À la fin, j'avais implémenté avec succès deux modèles de computer vision avec un troixième en cours.

Stagiaire en développement géomatique

Mai 2022 - Juil 2022
  • JavaScript
  • Node.js
  • Docker
  • PostgreSQL
  • Preuve de concept
  • A développé avec succès une preuve de concept pour une application web de routage en intérieur avec JavaScript/Node.js.
  • A connecté une application web front-end avec une base de données PostgreSQL roulant sur un container Docker Ubuntu.
  • A identifié les technologies les plus efficaces pour l'encodage des données géographiques, pour les libraires JavaScript/Node.js et quel engin de routage à utiliser.

Pour ce stage, j'ai été chargé de développer une preuve de concept pour une application web liée au routage en intérieur. Pendant le développement, j'ai du rechercher les technologies possibles et faire des décisions sur quelles technologies seraient les plus adaptées au projet. J'ai aussi du m'adapter à cause de problèmes de compatibilité que j'ai rencontré ce qui m'a poussé à utiliser des containers Docker pour le back-end de l'application.

L'application a été développée en JavaScript avec Yarn et Webpack. Pour le back-end, J'ai utilisé Docker Compose pour créer une base de données PostgreSQL, un API PostgREST et un engin de routage qui travailleraient tous les trois ensemble. L'appliation web a été deployée sur un serveur IIS.

À la fin du stage, l'application était totalement fonctionelle et répondait aux exigences du client.
