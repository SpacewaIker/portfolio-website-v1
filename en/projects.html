Museum Manager

Oct 2022 - Nov 2022
  • Java
  • JavaScript
  • Spring
  • PostgreSQL
  • GitHub Actions
  • Creating a webapp for managing a museum's items, employees, and customers.
  • Developing a modern webapp from the domain model design and the Java backend using Spring and a PostgreSQL database to the frontend using JavaScript.
  • Collaborating in a team of six developers using GitHub and a CI/CD pipeline through GitHub Actions.

This project is a management web application for a fictional museum that allows the management of the museum's items, employees, and customers. Items can be put in specific rooms or in storage and can be loaned by customers. Customers can also donate items to the museum. Employees can approve loans and donations in addition to putting items in and out of storage. Finally, the museum's owner can perform the same actions as the employees in addition to hiring and firing employees.

As this project is more or less a follow-up to the DiveSafe project, the goals are similar: to learn about project management techniques while familiarizing ourselves with the tools used in the industry. As such, we will use a Kanban board to manage our tasks and a CI/CD pipeline through GitHub Actions to automate the deployment of our application. We will also use GitHub to collaborate on the project and to manage our code. Our backend will be written in Java using the Spring framework connected to a PostgreSQL database. Finally, our frontend will be written in JavaScript.

DiveSafe

Jan 2022 - Apr 2022
  • Java
  • MVC
  • UML
  • TDD
  • BDD
  • Gherkin
  • JavaFX
  • Developed a desktop GUI application for managing diving members, trips, and equipment rental.
  • Collaborated with five other developers from the architecture design to the development and testing.
  • Used the Model-View-Controller design pattern and developed the application both as Test- and Behavior-Driven-Development.
  • Designed and created a logo for the application.

The DiveSafe desktop application is a management application for a fictional diving company. It can manage club members, diving guides and their schedules, member trips, and equipment rental.

The purpose of this project was to experiment with popular design pattern and development styles. As such, the application's architecture was based on the Model-View-Controller pattern. The Model was built from a UML diagram and the Java classes were automatically generated from a model description using the Umple tool. Some parts of the Controller were developed with Test-Driven-Development, while for others we used Behavior-Driven-Development. For the former, we used JUnit, and for the latter, we used Cucumber and Gherkin user stories. Finally, for the graphical user interface, we used JavaFX.

I also designed and created the application's logo which you can see above!

AirDraw

Oct 2021 - Nov 2021
  • Python
  • SciKit Learn
  • MediaPipe
  • AI/ML
  • Computer Vision
  • StreamLit
  • Built an artificial intelligence based web app in Python.
  • Built, trained, and saved the machine learning model used for the recognition of the hand signs.
  • Improved the model's performance through data augmentation techniques.

This project was the final assignment for the McGill Artificial Intelligence Society's "MAIS 202" AI bootcamp. The goal of this assignment was to use all that we learned throughout the bootcamp to come up with an AI-based application. My partner and I were interested in computer vision, and as such, decided to build an app where the user would draw shapes by waving their hand and making signs in front of a camera.

We handled the full development of the app, from the gathering of the training data, to choice of which machine learning algorithm to use, and the building and training of the model. We decided to go for a decision tree algorithm for the classification of the hand signs, and we used MediaPipe's Hands library to extract the hand's position from the video stream. For the web app, we used StreamLit and PyVirtualCam as these libraries were straightforward and corresponded perfectly to our needs.

As this was a team project, I developed my teamwork skills. Notably, I had to demonstrate good communication and time management skills due to some problems I encountered with my teammate. The code for the model training as well as a proof of concept web app are available on the GitHub repository. A DevPost page was also made for this project.

Pong

Oct 2021 - Dec 2021
  • Java
  • Game Development
  • Physics Simulation
  • Object-Oriented Programming
  • Recreated the Pong game in Java with a physics simulation for the ball's movement.
  • The game includes settings such as the opponent's difficulty, the physics simulation's speed, and the colour of the ball and paddles.

For this project I recreated the classic Pong game in Java. This was my first project in Java and a more formal introduction to Object Oriented Programming. As such, the game was first developed with a functional/imperative programming style, and then later converted to an Object-Oriented Programming style. This allowed for easier maintenance of the code, as well as additional features, such as multiple balls simulation through multithreading.

The game includes many features such as a scoring system with renamable players, colour settings for the ball and paddles, game settings, and keyboard shortcuts for actions like restarting the game, pausing the game, and quitting the game.
